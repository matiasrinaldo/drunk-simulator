# Phase 4 — UI Design Contract (Motion Contract)

> Visual and interaction contract for Phase 4 (Animaciones) of Drunk Simulator.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **PLATFORM NOTE — read first:** This is a **Unity 6000.3.11f1 game using URP 17.3**, not a
> web/React project. The standard UI-SPEC template (shadcn, fonts, px spacing scale, component
> libraries) is web-oriented and **mostly does not apply**. Web-only rows are marked
> `N/A — Unity Screen Space Overlay HUD (no web stack)`.
>
> **This phase adds NO new UI.** The HUD already exists and is built entirely by code in
> `Assets/_Project/UI/HUD/HUDController.cs` (a `RuntimeInitializeOnLoadMethod` singleton that
> creates a `Canvas` → `HUDGroup` → `DrunkBar` (`Track`+`Fill`) + `MoneyText`). Phase 4 layers
> **motion** onto these existing elements. Therefore this document is a **motion design contract**
> (timing, easing, amplitude, frequency, trigger conditions) rather than a static layout contract.

---

## Scope of This Contract

This contract governs the **HUD-side (Canvas) animation** for Phase 4. Two requirements:

| Req | What it animates | In this contract? |
|-----|------------------|-------------------|
| **ANIM-02** (animation by code, no AnimationClip) | Existing HUD elements (`DrunkBar`, `MoneyText`) modified per-frame in `HUDController` | **YES — fully specified below.** This is the primary subject of this UI-SPEC because it touches the Canvas. |
| **ANIM-01** (Animator state machine, ≥2 states + conditional transition) | See decision below | **Boundary only.** Document which option; full Animator/clip authoring belongs to RESEARCH.md + the plan, not this UI motion contract. |

### ANIM-01 — locked recommendation (world-space, OUTSIDE this Canvas contract)

Per `04-RESEARCH.md` (Open Question 1, primary recommendation) the recommended ANIM-01 target is a
**decorative door** in world space (a door that opens on approach **without** loading a scene,
avoiding the scene-load timing pitfall). A door is a "main visible element", satisfies ≥2 states
(`Closed`/`Open`) + a conditional transition (parameter `Open`), and **does not** live on the Canvas.

> **ANIM-01 is world-space and therefore outside the Canvas/HUD motion contract.** Its timing
> (~0.5–0.7 s open, ease-out hinge rotation 0°→~95°) and the Animator authoring are defined in
> `04-RESEARCH.md` and will be locked by the planner. A HUD-Animator alternative was **rejected**
> (the HUD is 100% code, built `BeforeSceneLoad`; injecting an Animator prefab breaks that pattern —
> RESEARCH "ANIM-01 Candidate Survey"). This UI-SPEC does **not** specify ANIM-01 motion values.

### ANIM-02 — locked decisions (THIS contract)

Two code-driven HUD animations, both modifying RectTransform / CanvasGroup / TMP properties in
`Update`/coroutine, **with zero AnimationClip**:

1. **Drunk-bar pulse** — the `DrunkBar` "breathes" (scale throb) at an amplitude/frequency scaled by
   `DrunkManager.EffectIntensity`. More drunk → stronger, faster pulse. Reinforces the game theme.
2. **Money-text wobble** — `MoneyText` shakes briefly when a purchase is rejected for insufficient
   funds (the `!CanAfford` rejection path + rejection SFX already exist in `PlayerPickup` from Phase 1).

## Motion Tokens (the real contract for this phase)

All values are **verifiable in Play Mode**. Durations in seconds; amplitudes are unitless scale
factors or local pixels; frequency in Hz (cycles/second). Use `Time.unscaledDeltaTime`-free,
standard `Time.time`/`Time.deltaTime` per project convention (no `Time.timeScale` manipulation in this phase).

### Motion Token A — Drunk-Bar Pulse (continuous, EffectIntensity-driven) — ANIM-02 primary

| Property | Value | Notes |
|----------|-------|-------|
| Target element | `DrunkBar` RectTransform **`localScale`** | Scale of the **bar container**, NOT the `Fill` and NOT `fillAmount`. See landmine below. |
| Pivot for scale | bar pivot stays `(0,0)` as built; pulse must scale around the **bar center** | Either temporarily scale around center or use a child wrapper so the throb reads symmetrically. Executor's call; the visible result must be a centered breathing, not a corner-anchored stretch. |
| Easing / waveform | **Sine** (`Mathf.Sin`) | Matches project's sinusoidal drunk-distortion convention (`PlayerMovement`, `MouseLook`, `CarController`). |
| Amplitude (scale delta) | **0.00 → 0.08**, lerped by `EffectIntensity` | `scale = 1.0 + Mathf.Sin(Time.time * 2πf) * amp`, `amp = Mathf.Lerp(0f, 0.08f, EffectIntensity)`. Sober ⇒ no pulse (amp 0). Fully drunk ⇒ ±8% throb. |
| Frequency | **0.0 → 1.8 Hz**, lerped by `EffectIntensity` | `f = Mathf.Lerp(0f, 1.8f, EffectIntensity)`. Slow heartbeat when tipsy, faster "pounding" when very drunk. Convert to angular: `Time.time * f * 2π`. |
| Activation threshold | `EffectIntensity > 0.05` | Below this, hold `localScale = Vector3.one` exactly (no jitter at rest). |
| Axis | Uniform XY (`new Vector3(s, s, 1f)`) | Z untouched (Overlay UI). |
| Per-frame source | Read `EffectIntensity` **every frame** in `HUDController.Update` | Follow the established project pattern: cache `DrunkManager` (re-bound in `OnSceneLoaded`, already present), read `EffectIntensity`, multiply. |

### Motion Token B — Money-Text Wobble (one-shot, event-driven) — ANIM-02 secondary

| Property | Value | Notes |
|----------|-------|-------|
| Target element | `MoneyText` RectTransform **`anchoredPosition`** (X shake) | Overlay UI ⇒ use `anchoredPosition`, never world `position` (RESEARCH Pitfall 5). |
| Trigger | **Purchase rejected for insufficient funds** | Fires off the existing `!CanAfford` branch in `PlayerPickup` (Phase 1), same moment the rejection SFX plays. Requires a one-shot signal into `HUDController` (e.g. a `public static` event/method like `HUDController.FlashMoneyRejected()` — exact wiring is the planner's call; the executor must not invent a new persistent store). |
| Waveform | Damped horizontal shake | `offsetX = Mathf.Sin(elapsed * shakeFreq * 2π) * amp * (1 - elapsed/duration)` |
| Easing | **Ease-out** (linear-decaying envelope) | Amplitude decays to 0 over the duration so it settles cleanly. |
| Amplitude (start) | **6 px** | Peak horizontal displacement at t=0, decaying to 0. |
| Frequency | **12 Hz** | Brisk, attention-grabbing shake. |
| Duration | **0.30 s** | Short; matches the rejection SFX feel. |
| Optional color flash | Tint `MoneyText.color` white→destructive red→white over the same 0.30 s | Optional reinforcement; if used, use the Destructive color below, return exactly to `Color.white`. |
| Reset guarantee | After the coroutine ends, set `MoneyText.anchoredPosition` back to its built value `(0, 28)` and color to `Color.white` | No drift after repeated rejections. |
| Re-trigger | Restart (or no-op) if already wobbling | Do not stack offsets; a fresh rejection restarts the single coroutine. |

## ⚠️ Landmine — Pulse must NOT fight the existing fill lerp

`HUDController.Update` **already** lerps `fillImage.fillAmount` toward `targetFillAmount`
(`Mathf.Lerp(fillImage.fillAmount, targetFillAmount, 3f * Time.deltaTime)`, with snapping at <0.001).

**Hard rule:** Motion Token A (the pulse) MUST operate on a **different property and a different
GameObject** than the fill lerp:

- Fill lerp owns: `Fill` Image → `fillAmount` (the data/value channel). **Do not touch.**
- Pulse owns: `DrunkBar` (the container) → `localScale` (a pure transform channel).

Because they write different properties on different objects, they cannot fight. **Never** drive the
pulse through `fillAmount` (that would corrupt the borracho reading) and **never** scale the `Fill`
child alone (the throb would distort the value bar). Verified element names/hierarchy:
`HUDGroup → DrunkBar (RectTransform, sizeDelta 220×20) → Track → Fill` in
`HUDController.BuildHUD()`.

## Interaction / State Contract

| State | Visual behavior |
|-------|-----------------|
| Sober (`EffectIntensity ≤ 0.05`) | `DrunkBar.localScale == Vector3.one`. No pulse. Fill lerp behaves exactly as today. |
| Drinking / drunk | `DrunkBar` throbs; amplitude & frequency rise with `EffectIntensity`. Fill continues lerping independently. |
| Purchase rejected (insufficient funds) | `MoneyText` wobbles 0.30 s (6 px → 0, 12 Hz), optional red flash, returns to `$N` white at `(0,28)`. |
| Scene change | Pulse re-binds via existing `OnSceneLoaded` `DrunkManager` re-bind; no special handling needed. Wobble coroutine, if mid-flight, must be safe across `DontDestroyOnLoad` (the HUD persists). |
| HUD hidden (`SetVisible(false)` on Result screen) | Both animations inert (the GameObject is deactivated by existing `SetVisible`). On re-show, reset to rest state. |

## Acceptance Checks (Play Mode, verifiable)

1. **Pulse off when sober:** Fresh scene, no drinks → `DrunkBar` is perfectly still (`localScale == (1,1,1)`).
2. **Pulse scales with drunkenness:** After several sips, the bar visibly throbs; throb is stronger
   and faster as `EffectIntensity` rises — distinguishable by eye without inspecting values.
3. **Pulse does not corrupt fill:** While pulsing, the fill level still lerps correctly to
   `NormalizedLevel`; the value bar is not distorted (pulse is on `DrunkBar.localScale`, not `Fill`/`fillAmount`).
4. **Wobble on rejection:** Try to buy a drink with insufficient money → `MoneyText` shakes ~0.30 s and
   settles back to `$N` at its original position and white color; coincides with existing rejection SFX.
5. **No drift:** After repeated rejections, `MoneyText` returns exactly to `anchoredPosition (0,28)`.
6. **No AnimationClip:** Both HUD animations are pure code (no `.anim` asset references) — satisfies ANIM-02.
